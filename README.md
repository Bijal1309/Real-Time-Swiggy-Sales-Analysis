# 📊 Swiggy Sales & Operational Intelligence Dashboard

## 🚀 Project Overview
This project delivers a comprehensive data analysis of Swiggy’s sales operations. By transforming raw, multi-dimensional transaction data into actionable business intelligence, this dashboard provides a 360-degree view of regional performance, customer satisfaction, and sales trends.
This repository demonstrates the end-to-end data pipeline, from cleaning messy raw datasets to extracting high-level KPIs that drive business strategy.

## 📈 Key Business Insights
The analysis focuses on answering critical business questions:
- Revenue Drivers: Which cities and food categories contribute most to the bottom line?
- Customer Sentiment: Is there a correlation between order frequency and average customer ratings?
- Trend Analysis: Identifying peak demand cycles (Daily, Weekly, and Monthly) to optimize delivery operations.

## 🛠 Technical Pipeline
- Data Preprocessing: Utilized Pandas for cleaning, handling null values, and normalizing inconsistent date formats.
- Feature Engineering: Derived critical metrics, including yearcorrect and revenue-per-order, to enable deeper trend analysis.
- Statistical Analysis: Performed correlation mapping to evaluate relationships between ratings and revenue.
- Visualization: Built a suite of professional charts using Matplotlib/Seaborn to communicate complex data patterns clearly.

## 📊 Key Performance Indicators (KPIs)
I have developed the following metrics to track business health:
- Total Sales (₹): Overall revenue generated from food orders.
- Average Rating: Customer satisfaction benchmark across all restaurants.
- Average Order Value (₹): Efficiency metric showing revenue per transaction.
- Ratings Count: Total customer engagement volume.
- Total Orders: Velocity metric for order fulfillment.
